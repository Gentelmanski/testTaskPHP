Описание задания
Разработать простое REST API для управления списком задач (To-Do List). Требуется реализовать CRUD-операции для задач с валидацией данных, используя PHP и SQLite. Код должен быть организован по шаблону MVC и загружен в репозиторий.

Обязательные операции:

POST /tasks – создание задачи (поля: title, description, status)

GET /tasks – получение списка всех задач

GET /tasks/{id} – получение одной задачи

PUT /tasks/{id} – обновление задачи

DELETE /tasks/{id} – удаление задачи

Требования:

Валидация: поле title не должно быть пустым.

Использование SQLite в качестве базы данных.

Код должен быть оформлен в стиле MVC (Model-View-Controller).

Приложить простой HTML-интерфейс для демонстрации работы API (опционально, но в решении предоставлен).

Решение
Технологии
PHP 7.4+ (без фреймворков, на чистом PHP)

SQLite (файл базы данных создаётся автоматически)

HTML5 + JavaScript (Fetch API) – простой клиент для управления задачами

MVC – разделение логики на модель (Task), контроллер (TaskController) и представление (frontend.html)

Структура проекта
text
project/
├── index.php          # точка входа, роутинг
├── Task.php           # модель задачи (работа с БД)
├── TaskController.php # контроллер (обработка запросов, валидация)
├── frontend.html      # пользовательский интерфейс
└── database.sqlite    # файл БД (создаётся при первом обращении)
Как это работает
Маршрутизация (index.php)
Все запросы направляются в index.php.

При обращении к корню (/ или /index.php) отдаётся статический HTML-файл (frontend.html).

Если URI начинается с /tasks, происходит анализ HTTP-метода и параметра id.

В зависимости от метода вызывается соответствующий метод контроллера.

Модель (Task.php)
Отвечает за взаимодействие с базой данных SQLite.

При первом обращении создаёт файл database.sqlite и таблицу tasks.

Методы:

all() – получить все задачи

find($id) – найти задачу по ID

create($data) – создать новую задачу

update($id, $data) – обновить существующую

delete($id) – удалить задачу

Использует PDO с подготовленными выражениями для безопасности.

Контроллер (TaskController.php)
Принимает запросы от маршрутизатора.

Валидирует входные данные (проверка title на пустоту).

Возвращает ответы в формате JSON с соответствующими HTTP-статусами.

Представление (frontend.html)
Простой интерфейс на HTML и JavaScript.

Отображает список задач, форму добавления, кнопки редактирования и удаления.

Взаимодействует с API через fetch().

API Endpoints
Метод	URL	Описание	Тело запроса (JSON)	Ответ (JSON)
GET	/tasks	Получить все задачи	–	Массив задач
GET	/tasks/{id}	Получить задачу по ID	–	Объект задачи или 404
POST	/tasks	Создать новую задачу	{"title": "..." , "description": "..." , "status": "..."}	{"message": "...", "id": id} (201)
PUT	/tasks/{id}	Обновить задачу	{"title": "..." , "description": "..." , "status": "..."} (любые поля)	{"message": "..."} (200) или ошибка
DELETE	/tasks/{id}	Удалить задачу	–	{"message": "..."} (200) или 404
Примечание: Поле status может принимать любые строковые значения (рекомендуется: pending, in_progress, completed). По умолчанию при создании – pending.

Валидация
При создании и обновлении (если передано поле title) проверяется, что оно не пустое.

При невалидных данных возвращается ответ с HTTP-статусом 422 и списком ошибок.

Запуск проекта
Убедитесь, что PHP установлен (версия 7.4 или выше) и доступен в командной строке.

Склонируйте репозиторий или скопируйте все файлы в папку проекта.

Запустите встроенный веб-сервер PHP:

bash
php -S localhost:8000
Откройте браузер по адресу http://localhost:8000.

Пользуйтесь интерфейсом – можно создавать, просматривать, обновлять и удалять задачи.

Тестирование API можно выполнить через curl или Postman:

bash
curl -X GET http://localhost:8000/tasks
curl -X POST http://localhost:8000/tasks -H "Content-Type: application/json" -d '{"title":"Купить молоко"}'
Особенности реализации
База данных SQLite создаётся автоматически при первом обращении к модели.

Для обновления задачи достаточно передать только изменяемые поля.

Код организован по принципу MVC, что облегчает дальнейшее расширение.

Интерфейс на фронтенде использует нативный JavaScript без дополнительных библиотек.
